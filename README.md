# Платформа управления волонтерами

Веб-платформа по ТЗ: регистрация по телефону, анкеты двух типов, модерация заявок, мероприятия,
координаторы, волонтерские часы и аналитика. Три роли: волонтер, координатор, администратор.

## Стек

- **Backend:** Node.js 18+, Express, SQLite (better-sqlite3), JWT, bcrypt
- **Frontend:** SPA без сборки (ES-модули, хеш-роутер), светлая и темная темы, адаптив
- **База:** один файл `data/app.db`, схема создается автоматически при первом запуске

## Запуск

```bash
npm install
cp .env.example .env        # задайте JWT_SECRET
npm run seed                # демо-данные: 40 волонтеров, 3 координатора, 6 мероприятий
npm start                   # http://localhost:3000
```

Полный сброс базы: `npm run reset`. Разработка с автоперезапуском: `npm run dev`.
Docker: `docker build -t volunteers . && docker run -p 3000:3000 volunteers`.

### Демо-доступы (пароль `password123`)

| Роль | Телефон |
|---|---|
| Администратор | +7 701 000 00 01 |
| Координатор | +7 701 000 00 02 |
| Волонтер | +7 702 100 00 00 |

При `SMS_PROVIDER=console` коды подтверждения печатаются в лог сервера и возвращаются
в ответе (поле `devCode`) — так регистрацию можно пройти без реального SMS-шлюза.
Для боевого запуска подключите провайдера в `src/services/sms.js`, функция `deliver()`.

## Что реализовано

**Авторизация.** Регистрация по номеру телефона с SMS-кодом, вход по телефону и паролю,
восстановление пароля через код, смена пароля в кабинете. Пароли — bcrypt, сессии — JWT.
Ограничение частоты запросов кода и попыток входа.

**Типы волонтерства.** Выбор после регистрации: организация (14+) или партийное крыло (18+).
Возраст проверяется на сервере, порог задается в `config.minAge`.

**Анкета.** Личные данные, образование, занятость, языки, направления, навыки, качества,
интересы, цели, мотивация, опыт, авто, ноутбук, свободное время. Поле `answers_json` принимает
произвольные дополнительные вопросы без изменения схемы — когда придет финальный список,
достаточно добавить их в форму.

**Модерация.** Статусы `draft → pending → approved | rejected | revision`. Администратор одобряет,
отклоняет или возвращает на доработку с комментарием. До одобрения мероприятия недоступны
(middleware `requireApproved`).

**Личный кабинет.** Профиль, статус заявки, часы со счетчиком, достижения, закрепленный
координатор, ближайшие мероприятия, история участия, журнал начислений, редактирование данных.

**Мероприятия.** Создание администратором: название, описание, дата, время, место, количество
волонтеров, координатор, требования, направления, статус. Волонтеры просматривают, записываются
и отменяют запись до дедлайна (`CANCEL_DEADLINE_HOURS`).

**Координаторы.** Список записавшихся с навыками и опытом, принятие в команду или отклонение,
роль в команде, отметка явки, начисление часов, комментарии, массовое закрытие мероприятия,
статистика своей команды.

**База волонтеров.** Полная карточка, фильтры (возраст, пол, город, статус, тип, навыки,
компетенции, направления, интересы, часы, мероприятия, координатор, дата регистрации),
быстрый поиск по ФИО и телефону, сортировка, пагинация, экспорт в CSV для Excel.

**Подбор.** Ранжирование кандидатов по совпадению навыков и направлений с бонусами за опыт,
активность, наличие авто/ноутбука и совпадение по свободному времени.

**Часы.** Координатор отмечает явку и часы, начисление автоматически уходит в профиль.
При исправлении данных начисление пересчитывается, а не задваивается.

**Аналитика.** Все метрики из ТЗ: волонтеры, новые заявки, активные, координаторы, проведенные
и предстоящие мероприятия, сумма и среднее часов, участники по направлениям, регистрации по
месяцам, топ волонтеров и координаторов.

**Управление пользователями.** Изменение данных, роль, блокировка, удаление, назначение
координаторов, сброс пароля, история активности.

**Уведомления.** Регистрация, одобрение, отклонение, назначение координатора, новое и измененное
мероприятие, подтверждение участия, начисление часов. Канал доставки один (внутренний);
Telegram и WhatsApp добавляются функцией в `CHANNELS` (`src/services/notifications.js`).

## Структура

```
src/
  server.js              точка входа, монтирование роутов, обработка ошибок
  config.js              настройки из .env
  db.js                  схема SQLite, представление volunteer_stats, журнал действий
  seed.js                демо-данные
  middleware/auth.js     JWT, проверка ролей, доступ после одобрения
  services/sms.js        коды подтверждения
  services/notifications.js  уведомления и шаблоны
  routes/
    auth.routes.js         регистрация, вход, восстановление
    profile.routes.js      кабинет, личные данные, уведомления
    applications.routes.js тип волонтерства, анкета, модерация
    events.routes.js       мероприятия, запись, отмена
    coordinator.routes.js  команда, явка, часы
    volunteers.routes.js   база, фильтры, подбор, экспорт
    users.routes.js        управление пользователями
    analytics.routes.js    дашборд
    dictionaries.routes.js справочники
    _serialize.js          сериализация ответов
  utils/                 хелперы и справочники навыков
public/                  index.html, app.js, styles.css
```

## Что стоит доработать перед боем

- Подключить реального SMS-провайдера и убрать `devCode` из ответа (`SMS_PROVIDER != console`).
- Задать длинный `JWT_SECRET` и запустить за HTTPS-прокси.
- Для нагрузки выше нескольких тысяч пользователей — перенести схему на PostgreSQL:
  запросы вынесены в роуты и переписываются без изменения структуры проекта.
- Заделы под модули из ТЗ уже есть: рейтинг и бейджи (`volunteer_stats`, `buildAchievements`),
  экспорт (CSV готов), QR-регистрация и сертификаты (таблица `registrations`), календарь
  (`events.starts_at`), Telegram и WhatsApp (`CHANNELS`), API для мобильного приложения (REST + JWT).
